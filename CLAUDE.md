# World Cup 2026 Live Dashboard

## What this project is

A public web dashboard showing live + historical FIFA World Cup 2026 data (scores, standings, fixtures, stats, per-match detail), built end-to-end on a **$0 budget**. It's video content for a YouTube series about building a live dashboard for free.

## Decisions already made (don't relitigate these without reason)

### Data source: ESPN public Site API
- Chosen after testing 8 APIs. Full reasoning in `docs/findings.md`.
- Base URL: `https://site.api.espn.com/apis/site/v2/sports/soccer/fifa.world/`
- Key endpoints:
  - `scoreboard` — all matches for a day, live scores, goal events (`competitions[0].details`).
  - `summary?event={id}` — full match detail incl. lineups, formations, in-game stats.
  - `standings/total` — group tables.
- Measured lag: **~1–2 minutes** behind real time (validated empirically — see `docs/findings.md`). This is 3–5× faster than football-data.org's free tier (~6 min), which was rejected.
- Caveat: ESPN's API is **undocumented, no SLA**. Endpoints could change. Wrap all ESPN calls in a thin adapter layer so a schema break is a one-file fix.

### Supporting sources
- Static fixtures/groups: `https://github.com/openfootball/worldcup.json` (load at build time).
- Post-match rich detail (xG, shot maps — v2 only): Sofascore unofficial API, on demand only. TOS-grey; documented in `docs/findings.md`.

### Architecture (no database)
```
[Cloudflare Workers] ← whole app (Next.js via OpenNext adapter)
   ├──> /                 static assets served from Workers Assets binding
   ├──> /api/live         Worker route, 30s edge cache, proxies ESPN
   └──> /history.json     static asset, regenerated by GitHub Actions cron every ~2h
```
- **No persistent database.** Live data = edge cache (30s TTL). Historical = static `history.json` in the repo, regenerated by cron, served as a static asset. Reasoning in `docs/findings.md` and `docs/deploy-storage-decision.md`.
- **No orchestration tool** (no Airflow). Just GitHub Actions cron + Cloudflare.
- ESPN gets hit ~2 times/minute per region regardless of user count, thanks to the edge cache.

### Tech stack
- **Next.js** (App Router) + **TypeScript** + **Tailwind CSS** + **TanStack Query** for data fetching/polling.
- Deploy via **Cloudflare Workers** using the **OpenNext adapter** (`@opennextjs/cloudflare`). Config in `wrangler.jsonc` and `open-next.config.ts`. Deploy on git push (CF auto-builds) or locally via `npm run deploy`.
- Charts (if needed): Recharts. Icons: Lucide.

## Scope — v1 (see `docs/requirements-v1.md` for full detail)

Five routes:
- `/` — Home: today's / upcoming matches with live scores. Click a card → match detail.
- `/standings` — group stage tables (12 groups of 4).
- `/fixtures` — all 104 matches, filterable by team/group/date.
- `/stats` — top scorers, top assisters, other individual leaders.
- `/match/[id]` — match header, goal timeline, lineups, in-game stats, event feed.

**Out of scope for v1:** accounts, predictions, notifications, video highlights, tactical viz, i18n, knockout bracket view.

**Known open risk:** ESPN may not expose a single tournament-leaders endpoint for `/stats`. May need to aggregate from per-match data or fall back to Sofascore. Treat `/stats` as soft-scope — defer to v2 if it blocks launch.
